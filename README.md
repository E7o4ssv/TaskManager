## Возможности

- **Авторизация** — регистрация и вход по email/паролю
- **Роли** — менеджер видит все проекты и управляет участниками/ролями; работник видит только проекты, в которые его добавили
- **Должности** — менеджер может создавать должности (разработчик, дизайнер и т.д.) и назначать их пользователям
- **Чат** — общий канал и личные переписки
- **Проекты и задачи** — канбан по статусам, приоритет, срок, исполнитель, прикрепление документа к задаче
- **Файлы** — по проектам: загрузка и скачивание, список с автором и датой
- **Приглашения по ссылке** — менеджер создаёт ссылку на проект; по ссылке можно войти/зарегистрироваться и присоединиться к проекту

## Запуск

1. Установите зависимости:
   ```bash
   npm install
   ```

2. Создайте файл `.env` (можно скопировать из `.env.example`):
   ```
   DATABASE_URL="file:./dev.db"
   ```

3. Примените схему БД:
   ```bash
   npx prisma generate
   npx prisma db push
   ```

4. Запустите сервер:
   ```bash
   npm run dev
   ```

5. Откройте [http://localhost:3000](http://localhost:3000), зарегистрируйтесь и войдите.
